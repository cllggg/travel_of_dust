# 《灰尘的旅行》全书有声书制作教程

> 本教程记录如何**用自己的声音 + 本地 TTS 模型**为《灰尘的旅行》全书 62 章生成**有感情、有节奏、生动**的中文有声书。
>
> 目标读者：10–20 岁青少年  
> 目标硬件：Mac mini M4 Pro 48GB（本地运行，**无需云端**）  
> 核心方案：**自录声音 → 训练个人声模 → 批量合成 62 章 MP3**

---

## 一、为什么选"自录 + TTS 克隆"

| 方案 | 优势 | 劣势 |
|---|---|---|
| 逐章直接朗读 | 情感最真实 | 62 章 × 15 分钟 ≈ **15 小时+**，状态不一、容易失误 |
| 直接用现成 TTS | 全自动 | 不是你的声音，青少年缺少亲切感 |
| ✅ **自录 + TTS 克隆** | **一次录制、永久复用、风格统一、效率与质量兼得** | 需一次性投入录音与训练 |

**结论**：以你的硬件条件，**录制 30–60 分钟干净语音 → 训练个人 TTS → 全自动生成 62 章**，是性价比与质量最高的最优解。

## 四、完整操作流程（5 步）

### 第 1 步：环境准备（一次性，约 30 分钟）

```bash
# 1. 安装 Homebrew（如未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 2. 安装 Python 与 ffmpeg
brew install python@3.11 ffmpeg git

# 3. 克隆 GPT-SoVITS（推荐一键整合包）
git clone https://github.com/RVC-Boss/GPT-SoVITS.git
cd GPT-SoVITS

# 4. 用 conda 创建独立环境（避免依赖冲突）
conda create -n SoVits python=3.10 -y
conda activate SoVits

# 5. 安装依赖（M 系列 GPU 加速）
pip install -r requirements.txt
pip install torch torchvision torchaudio
```

### 第 2 步：录制"声音素材"（一次性，约 1 小时）

> **整个方案最关键的一步**，决定最终克隆质量。

#### 🎤 硬件要求
- **麦克风**：推荐 Blue Yeti / 罗技 Blue Snowball / 铁三角 AT2020USB+（USB 麦，即插即用）
- **监听耳机**：推荐 铁三角 M30x / 索尼 MDR-7506
- **录音环境**：安静小房间 + 床上铺毛毯 / 衣柜内录音（消除回声）
- **采样率**：48kHz / 16bit
- **保存格式**：WAV（无损）

#### 📜 录制内容（推荐 30–60 分钟）
1. **朗读任意中文文本**（10–20 分钟）：新闻、散文、童话均可
2. **朗读本书片段**（10–20 分钟）：从《灰尘的旅行》第一、二、三部各选几段
3. **不同情感样例**（10 分钟）：
   - 平静叙述（"菌儿今天很快乐……"）
   - 兴奋好奇（"啊哈，原来是这样！"）
   - 轻柔温暖（"我们一起看看……"）
   - 严肃警示（"同学们要注意……"）
4. **多样句式**：疑问、感叹、长句、短句都要有

#### 🎙️ 录音操作
- 推荐工具：Mac 自带 **语音备忘录**（简单）或 **Audacity**（专业、免费）
- 距离麦克风 **15–20cm**
- **不要喷麦**，可加防喷罩
- 中途说错不用停，重复一遍正确版本，后期切掉即可

### 第 3 步：训练你的专属声模（约 2–4 小时）

1. 把录音文件放到 `GPT-SoVITS/data/your_name/raw/`
2. 一键预处理：

```bash
python scripts/auxiliary/extract_feature.py
```

3. 启动 WebUI：

```bash
python webui.py
```

4. 浏览器打开 `http://127.0.0.1:9874`
5. 按流程 **数据预处理 → 训练 GPT → 训练 SoVITS**
6. **M4 Pro 48GB 训练 30 分钟干净素材约 1–2 小时**
7. 训练完成后自动保存模型到 `GPT_weights/` 与 `SoVITS_weights/`

### 第 4 步：批量生成 62 章有声书（核心自动化）

启动 GPT-SoVITS API 服务（默认 `http://127.0.0.1:9880`），然后在项目根目录执行：

```bash
python3 generate_audiobook.py
```

**脚本行为**：
- 自动读取 `chapters_v2.json` 拿到每章 Markdown
- 解析章节标题与正文，**按段拆分**语音
- 调用本地 GPT-SoVITS API
- 支持**按句注入情感参考音频**（平静/兴奋/温暖/严肃）
- 产出 MP3 自动按章命名 `audio/part1-ch01.mp3` … `part3-ch31.mp3`
- 单章 10–20 分钟，全书预计 **6–10 小时**生成完毕（可后台挂着）

### 第 5 步：后期合成（可选，但强烈建议）

为做"有节奏、生动"的有声书，建议用 **Audacity**（免费）做：

1. **背景音乐**：选择轻音乐 / 古风 / 自然环境音（音量 ≤ 朗读声的 20%）
2. **段落间隙**：每段间加 **0.5–1.0 秒** 静音，听感更舒服
3. **章节头**：每章开头插入 **3 秒"叮"提示音**
4. **微调节奏**：个别字吞字/卡顿时微调
5. **统一响度**：导出时用 `→ 16-bit WAV → FFmpeg 转 MP3`（恒定码率 128kbps 足够）

## 五、本地资源消耗参考（M4 Pro 48GB）

| 阶段 | 内存占用 | 速度 |
|---|---|---|
| 预处理（特征提取） | ~3 GB | 10 分钟素材约 2 分钟 |

| 推理 1 章 | ~3 GB | 约 5–8 分钟 / 章 |

**M4 Pro 48GB 跑这套完全没压力**，全程本地、无网、无云端费用。

## 六、备选方案（按需选用）

### 方案 B：Mac 原生"说书人"快捷版（30 分钟可用）
用 macOS 自带 `say` 命令 + 中文语音包：

```bash
say -v "Tingting" -f 章节文本.txt -o 第一章.aiff
ffmpeg -i 第一章.aiff -b:a 128k 第一章.mp3
```

> ⚠️ 缺点：不是你的声音、机械感强、情感平淡。**不推荐用于正式作品**，但可作为"占位版"先跑通流程。

### 方案 C：云端兜底（最省心，质量最高）
若希望零训练、立刻有高质量，可考虑：
- 阿里 CosyVoice 在线 API（中文最优）
- ElevenLabs（英文为主）
- 字节豆包语音 / 腾讯云 TTS（中文好）

每月 ¥30–100 即可把全书跑完，但**不是你的声音**。
